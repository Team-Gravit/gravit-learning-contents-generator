# 검수 루브릭 (R1~R6)

이 파일은 reviewer 서브에이전트의 채점 **spec**이다. 항목별 1~5점 정수 채점 후 PASS/REJECT을 판정한다. 재시도 루프에서도 동일한 기준을 적용한다(완화 없음).

---

## 적용 대상

- **문제 단위 (R1~R5):** 생성된 각 문제에 대해 채점. 문제별 PASS/REJECT 판정.
- **레슨 단위 (R6):** 6문제 전체를 묶어 난이도 균형을 1회 채점. 레슨별 PASS/REJECT 판정.

"개념 편중 방지"는 **generator 생성 단계의 제약**으로 처리한다(루브릭 항목이 아님). `spec/learning-content-rules.md` 참조.

---

## 항목

| 항목 | 코드 | 적용 대상 | 설명 |
|---|---|---|---|
| 정답 정확성 | R1 | 문제(객/주) | 리뷰어가 직접 풀이한 결과와 제시 정답의 일치 여부 |
| 선지 변별력 | R2 | 문제(객) | 오답 선지의 그럴듯함 + 정답 선지의 유일성 |
| 난이도 적절성 | R3 | 문제(객/주) | 타겟층(CS 배경 취준생/학부생) 수준 적합도 |
| 발문-본문 정합성 | R4 | 문제(객/주) | 발문이 묻는 바와 본문 맥락의 자연스러움 |
| 해설 충실도 | R5 | 문제(객/주) | 정답 근거 + 오답 이유 설명의 학습 효과 |
| 난이도 균형 | R6 | 레슨 | 6문제 전체의 난이도 분포가 타겟층에 적합한지 |

---

## 채점 기준 (정수 1~5)

```
R1. 정답 정확성
  5: 풀이 결과와 제시된 정답 완전 일치, 해설 논리도 정확
  4: 정답 일치, 해설에 사소한 표현 부정확 (오개념 아님)
  3: 정답 일치, 해설 논리에 빈약한 부분 존재
  2: 정답이 모호하거나 특정 해석에서만 성립
  1: 정답이 명백히 오류

R2. 선지 변별력 (객관식만)
  5: 오답 선지 모두 그럴듯하되 명확한 오개념 기반, 정답이 유일
  4: 대부분 적절, 1개가 너무 쉽게 소거됨
  3: 2개 이상이 명백히 틀려 소거법으로 쉽게 풀림
  2: 복수정답 가능 선지 존재
  1: 선지 구성 무의미 또는 정답 선지 오류

R3. 난이도 적절성
  5: 개념 이해 + 약간의 응용 필요, 타겟층에 최적
  4: 적절하나 약간 쉽거나 어려움
  3: 단순 암기로 풀리거나 상위 10%만 풀 수 있는 수준
  2: 지나치게 쉬움(정의만) 또는 지나치게 어려움(대학원 수준)
  1: 타겟층과 완전 불일치

R4. 발문-본문 정합성
  5: 발문과 본문이 완벽히 일치, 자연스러운 문장
  4: 정합하나 사소한 어색함 (조사, 어미 등)
  3: 이해 가능하나 연결이 느슨
  2: 발문이 모호하여 의도 파악 어려움
  1: 발문과 본문이 논리적으로 불일치

R5. 해설 충실도
  5: 왜 정답인지/왜 오답인지 핵심 개념으로 설명, 학습 효과 높음
  4: 정답 근거 충실, 오답 해설 다소 부족
  3: 해설이 피상적 ("~이므로 정답이다" 수준)
  2: 핵심 개념 언급 없이 결론만 제시
  1: 해설 누락 또는 오개념 포함

R6. 난이도 균형 (레슨 단위)
  5: 6문제가 개념 이해~응용 범위에 고르게 분포, 타겟층에 최적
  4: 분포 적절하나 1~2문제가 약간 쉽거나 어려움
  3: 전반적으로 한쪽(쉬움/어려움)으로 치우침, 학습 흐름에 무리 있음
  2: 대다수가 타겟층 밖 난이도 (암기 일변도 또는 대학원 수준 일변도)
  1: 레슨 전체가 타겟층과 불일치
```

---

## 판정

### 문제 단위 (R1~R5)

PASS 조건 (전부 충족):
- R1 ≥ 4
- 적용 항목 평균 ≥ 3.5
- 어떤 항목도 2점 이하가 아닐 것 (최솟값 ≥ 3)
- 주관식: R2 제외한 4개 항목(R1, R3, R4, R5)으로 판정

REJECT (PASS 부정, 아래 중 하나라도 해당):
- R1 ≤ 3
- 적용 항목 평균 < 3.5
- 어떤 항목이든 2점 이하

### 레슨 단위 (R6)

- PASS: R6 ≥ 3
- REJECT: R6 ≤ 2 → 기존 문제를 보존한 채 **난이도만 조정**. generator 재호출 시 `lesson.improvement_direction`에 방향 명시(쉽게/어렵게/분포 조정)하고, 문제별로 어떻게 조정할지(맥락 보강, 선지 단순화, 응용 요소 추가 등)를 함께 전달

### 재시도

- 문제 단위 REJECT → 해당 문제만 재생성 (최대 3회)
- 레슨 단위 REJECT → 기존 문제 6개를 보존하되 **난이도만 조정하여 재생성** (발문·본문·선지를 새로 쓰지 말고, 너무 어려운 문제는 맥락·예시 보강 또는 선지 단순화, 너무 쉬운 문제는 응용 요소 추가). 최대 3회, 레슨 재시도 카운터는 문제 재시도 카운터와 별도
- 3회 초과 시 `manual-review` 태깅

## 출력 스키마

```json
{
  "lesson": {
    "R6": 4,
    "verdict": "PASS",
    "reject_reason": null,
    "improvement_direction": null
  },
  "problems": [
    {
      "problem_ref": "p1",
      "problem_type": "OBJECTIVE",
      "scores": { "R1": 5, "R2": 4, "R3": 4, "R4": 5, "R5": 4 },
      "avg": 4.4,
      "verdict": "PASS",
      "reject_reasons": null,
      "improvement_direction": null
    },
    {
      "problem_ref": "p5",
      "problem_type": "SUBJECTIVE",
      "scores": { "R1": 4, "R3": 2, "R4": 4, "R5": 3 },
      "avg": 3.25,
      "verdict": "REJECT",
      "reject_reasons": {
        "R3": "단순 용어 정의 수준으로 타겟층 대비 지나치게 쉬움",
        "avg": "평균 3.25로 기준(3.5) 미달"
      },
      "improvement_direction": {
        "R3": "개념 정의뿐 아니라 상황 적용을 묻도록 본문에 구체적 시나리오 추가"
      }
    }
  ]
}
```

- `reject_reasons`·`improvement_direction`은 **항목별 구조화 객체**. generator 재호출 시 프롬프트에 항목별로 삽입 가능해야 한다.
- 평균 기준 미달 같은 **합산 사유**는 `avg` 키에 담는다.
- PASS 문제는 `reject_reasons`·`improvement_direction`을 `null`로 둔다.

## R1 판정 프로토콜

리뷰어는 제시된 정답·해설을 **먼저 보지 않고** 발문·본문만으로 내부 추론하여 풀이한 뒤, 그 결과를 제시 정답과 대조하여 R1을 채점한다. 풀이 과정은 출력 스키마에 포함하지 않는다(내부 추론).
