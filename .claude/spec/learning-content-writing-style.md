# 학습 콘텐츠 표기·용어 스타일 규칙

## 적용 대상 필드

`lesson.sql`의 아래 문자열 리터럴에 적용한다.

- `lesson_staging.title`
- `problem_staging.instruction` / `content`
- `option_staging.content` / `explanation`
- `answer_staging.content` / `explanation`

SQL 키워드·컬럼명·ID·`label` 값은 해당 없음.

---

## S1. 한국어 맞춤법·띄어쓰기

- 표준 맞춤법을 지켜라.
  - `데이타` → `데이터`, `역활` → `역할`, `되요` → `돼요`
  - `틀리다`(잘못 선택됨)와 `다르다`(상이함) 구분
- 명백한 띄어쓰기 오류를 피하라.
  - `할수있다` → `할 수 있다`
  - `프로세스간` → `프로세스 간`
- 조사·어미 선택을 정확히 하라.
  - `~로써`(수단) / `~로서`(자격) 구분
  - `~에`와 `~에서`의 혼동 주의

## S2. CS 도메인 외래어 표기

표준 외래어 표기법을 따른다. 자주 틀리는 것:

| 금지 | 올바른 표기 |
|---|---|
| 해쉬 | 해시 |
| 캐쉬 | 캐시 |
| 메세지 | 메시지 |
| 쓰레드 | 스레드 |
| 알고리듬 | 알고리즘 |
| 데이타 | 데이터 |
| 네트웍 | 네트워크 |
| 서브넷트 | 서브넷 |
| 엑세스 | 액세스 |
| 컨텐츠 | 콘텐츠 |
| 비지니스 | 비즈니스 |
| 어플리케이션 | 애플리케이션 |

위 표는 고빈도 예시다. 확신이 없는 외래어는 표준 표기를 따른다.

## S3. 용어 일관성 (한 레슨 내)

- **같은 개념을 한 레슨 안에서 혼용하지 마라.** 예: 한 문제에서 `JavaScript`, 다른 문제에서 `자바스크립트` → 둘 중 하나로 통일한다.
- 영문/한글 표기는 레슨 단위로 하나를 고른다. 레슨마다 다른 선택은 무방하나, 한 레슨 내부에서는 고정한다.
- 약어와 풀네임은 최초 등장 시 `풀네임(약어)` 형태를 권장한다. 강제는 아니지만 선택한 형태는 유지한다.
- 6문제를 한 번에 쓰는 시점에 전체 가시성이 있으므로, 쓰기 직후 **전체를 훑어 일관성을 점검**한다.

---

## S4. CS 용어 적절성

- 문맥에 맞는 용어를 선택하라.
  - 잘못된 예: "프로세스가 스레드를 상속한다" (상속은 OOP 개념에 쓰는 말)
- 은어·구어를 쓰지 마라.
  - 금지: `디비`, `자스`, `셤` 등 → `데이터베이스`, `JavaScript`, `시험`
- 개념적으로 부정확한 표현을 피하라. (예: "TCP는 UDP보다 항상 빠르다")

---

## S5. 타이핑·스펠링

- 자모 누락·중복을 확인하라. (예: `구현해한다`, `알고리즘은은`)
- 영문 고유명사의 철자를 정확히 쓰라. (예: `JavaScript`, `PostgreSQL`, `Kubernetes`)

---

## 주의

- 본문에 코드 블록(` ```java ... ``` ` 등)을 포함할 때, 코드 블록 내부의 키워드·식별자는 맞춤법 규칙의 대상이 아니다.
- SQL 문자열 리터럴 내부에 작은따옴표가 필요하면 `''`로 이스케이프한다. 스타일 규칙은 이스케이프 전의 원문 기준으로 적용한다.
